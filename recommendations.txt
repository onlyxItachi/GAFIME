GAFIME ARCHITECTURAL RECOMMENDATIONS & REVIEW SUMMARY
=====================================================
Date: 28 December 2025
Role: Senior Code Reviewer / Architect

1. ARCHITECTURAL PHILOSOPHY
---------------------------
- GAFIME is a diagnostic engine, not a model trainer. The "Fail Fast, Fail Explicitly" approach must be preserved.
- Success is defined as providing a stable, multi-metric diagnostic of feature-target relationships.

2. C++ BACKEND (GAFIME_CORE) RECOMMENDATIONS
--------------------------------------------
- BATCHING: Avoid per-combination round-trips between Python and C++. The C++ layer must handle the loop over combinations (implemented via `score_combos`).
- CONCURRENCY:
    * Always release the Python GIL (`py::gil_scoped_release`) before entering heavy C++ computation.
    * Use OpenMP with `schedule(dynamic)` for the main combination loop to account for the varying cost of k-order interactions.
    * Ensure thread-safety by declaring scratch buffers (interaction vectors, rank data, MI histograms) inside the parallel region.
- MODERN C++23 STANDARDS:
    * Replace raw pointer arithmetic with `std::span` and lightweight 2D views (`Span2D`) for safety and readability.
    * Leverage stack-based allocation (e.g., `std::array`) for small-count histograms in Mutual Information calculations to reduce cache pressure.
- MEMORY MANAGEMENT:
    * Minimize heap allocations inside the "hot" combination loop. Pre-allocate or use thread-local scratchpads.

3. PYTHON INTEGRATION RECOMMENDATIONS
-------------------------------------
- BACKEND RESOLUTION: The Python `GafimeEngine` should detect the presence of `gafime_core` and prefer it over the pure-NumPy fallback.
- DATA PREPARATION: Ensure `X` and `y` are coerced to C-contiguous `float64` arrays before passing to the C++ core to avoid silent copies or stride errors.

4. FUTURE OPTIMIZATIONS
-----------------------
- SIMD: Explore explicit SIMD vectorization for the interaction product loop if interaction depth (k) increases.
- STABILITY: Current Spearman ranking uses `std::stable_sort`; evaluate if `std::sort` provides sufficient consistency for ties while improving performance.
- ASYNC: Consider asynchronous report generation where the UI/CLI stays responsive while the engine processes large feature sets.

STATUS: Core C++ implementation approved. Proceeding to system-wide integration.
