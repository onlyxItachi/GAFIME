# GAFIME Time Series Limitation Report

## Issue Identified
**Date:** 2026-01-20
**Dataset:** ING Hubs Türkiye Datathon (Churn Prediction)

## The Problem

GAFIME was designed for **tabular data** where:
- Each row is an independent observation
- Features are numerical or categorical
- No temporal dependencies between rows

However, the ING dataset has **time series data**:
- `customer_history.csv`: 5M+ rows with daily transactions per customer
- Temporal patterns (trends, seasonality) carry the churn signal
- GAFIME's current architecture cannot extract these patterns

## What Happened

1. Aggregating time series to 1 row per customer **destroyed** temporal information
2. GAFIME found features with ~22% correlation (similar to original)
3. Models didn't improve because time patterns were lost in aggregation

## What's Missing in GAFIME

### 1. Rolling Window Feature Engineering
GAFIME needs a **pre-processing step** that creates:
- 7-day, 14-day, 30-day, 60-day, 90-day, 180-day, 360-day rolling averages
- Rolling standard deviations
- Trend features (slope over time)

### 2. Strategy Categorization (New Concept)
Create categorical "strategy" features from rolling windows:
- **Short-term strategy**: based on 7, 14, 30-day features
- **Mid-term strategy**: based on 60, 90-day features  
- **Long-term strategy**: based on 180, 360-day features

### 3. Categorical Handling
GAFIME needs to:
- Accept categorical strategy labels
- Convert them to numerical encodings
- Then apply feature interactions

## Proposed Solution

Add a **Time Series Pre-Processor** to GAFIME that:
1. Takes raw time series data
2. Creates rolling window features per entity (customer)
3. Generates strategy categories
4. Outputs tabular data ready for GAFIME core

## Files Affected
- `gafime/preprocessors/` (NEW - time series module)
- `gafime/backends/fused_kernel.py` (if categorical handling added)

## Priority
**HIGH** - This is a fundamental limitation for any dataset with temporal data.
